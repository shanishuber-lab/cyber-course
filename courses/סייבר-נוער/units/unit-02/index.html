חלק 1

מהו פורט?

פורט = מספר שמזהה אפליקציה ספציפית במכשיר. טווח: 0 – 65,535 ▾ למה 65,535?
פורט הוא מספר של 16 סיביות (bit) — כלומר 2¹⁶ = 65,536 אפשרויות (0 עד 65,535). כל מספר יכול לשמש לשירות אחר. בפועל, רוב השירותים הנפוצים מרוכזים ב-0 עד 1023.
אנלוגיה — בניין דירות
כתובת הבניין = כתובת IP
מספר הדירה = פורט
חבילה לדייר ספציפי = מידע לאפליקציה

HTTPפורט 80
▾
HyperText Transfer Protocol — גלישה לאתרים. לא מוצפן — כל הנתונים נשלחים "גלויים". כיום מרבית האתרים עוברים ל-HTTPS.
HTTPSפורט 443
▾
HTTP Secure — גלישה מוצפנת. כשרואים 🔒 בדפדפן, השרת שומע על פורט 443. Instagram, Facebook, TikTok — הכל על 443.
FTPפורט 21
▾
File Transfer Protocol — העלאת קבצים רבים לשרת. ישן ולא מוצפן. SFTP (דרך SSH) הוא הגרסה המאובטחת המודרנית.
SSHפורט 22
▾
Secure Shell — חיבור מרחוק מאובטח לשרת עם שם משתמש וסיסמה. מנהלי מערכת משתמשים בו יומיום. פורט 22 הוא יעד נפוץ לניסיונות פריצה.
DNSפורט 53
▾
Domain Name System — "ספר הטלפונים" של האינטרנט. מתרגם www.instagram.com לכתובת IP. בלעדיו, היינו צריכים לזכור מספרים במקום שמות.

TCP Handshake — איך מתחברים?
לחיצת יד — TCP 3-Way Handshake
לפני כל שיחת TCP יש "לחיצת יד" בת 3 שלבים:

SYN — לקוח: "רוצה להתחבר?"
SYN-ACK — שרת: "בסדר, נתחבר!"
ACK — לקוח: "מתחיל לשלוח."

רק אחרי שלושת השלבים מתחילה העברת הנתונים האמיתית.

סיכום

מה למדנו?

פורט — מספר (0–65535) שמזהה אפליקציה ספציפית. IP = בניין, פורט = דלת לחדר.
פורטים מוכרים — HTTP:80 · HTTPS:443 · SSH:22 · FTP:21 · DNS:53
TCP — אמין, מסודר, בודק הגעה. מתאים לאתרים, הורדות, בנקאות.
UDP — מהיר, לא בודק. מתאים למשחקים, Zoom, Spotify.
Socket = IP + פורט. מזהה חיבור ספציפי באופן ייחודי — גלוי ב-netstat.
